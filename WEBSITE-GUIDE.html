Technical Stack

Your website uses three proven, widely-used technologies. No proprietary software, no subscriptions, no vendor lock-in. Any front-end developer can pick up and extend this codebase immediately.

HTML5

All 12 pages are standard HTML5 files. Open any file directly in a browser — no build step, no compilation, no local server required. This makes the site easy to edit and deploy.

Tailwind CSS v3 (CDN)

A utility-first CSS framework loaded from a CDN. Provides all the layout, spacing, color, and responsive design classes used throughout the site. No installation needed.

Alpine.js v3 (CDN)

A lightweight JavaScript framework (15KB) used for interactive elements: mobile menu toggling, FAQ accordions, the Dosha Quiz, and the Program Finder. No framework knowledge required to edit content.

Browser compatibility: Tested and functional on Chrome, Safari, Firefox, and Edge. Works on Windows, macOS, Linux, Android, and iOS. No polyfills or special configurations needed.

Section 3
15 New Features Explained

Beyond the standard pages and content, 15 purpose-built features have been added to improve conversions, build credibility, and make your website stand out from other Ayurveda centers. Each feature is documented below with what it does, where it appears, and how your team can update or activate it.

Interactive Features

A1

Dosha / Prakriti Quiz

Homepage — between "Who Is This For?" and "Featured Programs"

What it does: A 5-question interactive quiz. Visitors answer questions about their body type, digestion, stress response, sleep patterns, and health goals. Based on their answers, the system calculates whether they are dominant Vata, Pitta, or Kapha and recommends a specific Vaidyagrama program best suited to their constitution.

How it works: Pure JavaScript using Alpine.js. No backend, no server, no database. Results appear instantly in the browser. Works offline.

How to update it: The quiz logic lives in the doshaQuiz() function near the bottom of index.html. You can edit question text, answer options, result descriptions, and program recommendations directly in that function. No programming knowledge required — just edit the text strings.

A2

Program Finder Tool

Homepage — between "Featured Programs" and "Senior Medical Team"

What it does: A 4-step guided decision tool. Visitors answer: (1) What brings them to Vaidyagrama, (2) How long they can stay, (3) Their primary health goal, and (4) Their prior Ayurveda experience level. The tool recommends the most appropriate program with a clear "Book Free Chat" call-to-action.

How it works: Alpine.js component — all logic runs client-side. The recommendation algorithm matches visitor inputs against program profiles and returns the best fit.

How to update it: Open index.html and find the programFinder() function. The matching rules are in the calcPf() sub-function. To add a new program, add it to the programs array and add matching rules in the calculation logic.

A3

Animated Statistics Counters

Homepage — the dark Forest Green trust bar

What it does: When a visitor scrolls down to the trust bar, the numbers (180, 16, 80, 5, 97) animate from zero up to their final value over 2 seconds. This creates a dynamic, attention-holding effect that gives the statistics more perceived weight.

How it works: Uses the browser's built-in IntersectionObserver API to detect when the trust bar enters the viewport, then triggers a JavaScript counter animation. No external library needed.

How to update the numbers: In index.html, find each number in the trust bar — they are stored as data-target="180" attributes. Change the number in the attribute and the animation will automatically adapt. Also update the label text below each counter.

How to Update Content — Step by Step

You do not need a developer to update your website's text, photos, contact information, or pricing. Everything can be done with a basic text editor. This section walks through the most common update tasks.

How to Change Text

All website text lives directly in the HTML files. There is no content management system (CMS) or admin panel — the HTML file is the content. This means you can update text in under 2 minutes using any text editor.

Recommended free text editors: VS Code (code.visualstudio.com — best option), Notepad++ (notepad-plus-plus.org — Windows only, lightweight), or even the built-in Windows Notepad for simple changes.
  1. Open the relevant HTML file in your text editor. For example, to change homepage text, open index.html.
  2. Press Ctrl+F (Windows) or Cmd+F (Mac) to open the Find dialog. Type the text you want to change and press Enter — the editor will jump to that location in the file.
  3. Edit the text between the HTML tags. For example, to change a headline: find <h2>Our Senior Doctors</h2> and change "Our Senior Doctors" to your preferred wording. Do not delete or change the tags themselves (<h2> and </h2>).
  4. Save the file with Ctrl+S.
  5. Open the file in your web browser to preview the change. Double-click the HTML file or drag it into a browser window.
  6. If satisfied, upload the updated file to your web host to make the change live (see Section 6).

Hosting Your Website

Your website is a set of standard HTML files — it can be hosted on any web server in the world. Below are three options, presented from most to least recommended. Read Option A first; it takes 5 minutes and is completely free.

Option A — Netlify (Strongly Recommended)

Netlify is a professional static website hosting platform used by millions of websites worldwide. It is free for static HTML websites with no limitations on traffic. It is faster, simpler, and more powerful than Weebly for custom-coded sites.

RECOMMENDED

Netlify — Free, Fast, Professional

Takes approximately 5 minutes. No credit card required. Your site will be live at a *.netlify.app URL immediately, and you can connect your custom domain (e.g., vaidyagrama.com) within 30 minutes.

  1. Go to netlify.com and create a free account using your email address.
  2. After logging in, you will see your Netlify dashboard with a large dashed upload area that reads "Drag and drop your site folder here."
  3. Open File Explorer on your computer and navigate to your Desktop. You will see the vaidyagrama-rebuild folder.
  4. Drag the entire vaidyagrama-rebuild folder and drop it into the Netlify upload area in your browser.
  5. Netlify will upload all 12 pages and supporting files. After 30–60 seconds, your site will be live at a temporary URL like random-name-12345.netlify.app.
  6. Test your site by visiting that URL — click through all 12 pages and check that navigation, images, and interactive features work correctly.
  7. To connect your custom domain: in Netlify, go to Site Settings → Domain Management → Add Custom Domain. Enter your domain name (e.g., vaidyagrama.com).
  8. Netlify will show you DNS records to configure. Log in to wherever your domain is registered (e.g., GoDaddy, Namecheap, Weebly) and update the DNS nameservers to Netlify's: dns1.p01.nsone.net and dns2.p01.nsone.net. Alternatively, add the A record and CNAME values Netlify provides.
  9. DNS changes take 24–48 hours to propagate worldwide. After that, your custom domain will point to your new Netlify-hosted site.
  10. Netlify automatically provides a free SSL certificate (HTTPS) for your domain within minutes of DNS connecting. No configuration needed.

How to Update the Live Site After Launch

Every time you update an HTML file or add a new image, you need to re-deploy to make the changes live. There are three ways to do this with Netlify:

Method 1 — Drag & Drop (Simplest)

Log into Netlify → click your site → go to the Deploys tab → drag the updated vaidyagrama-rebuild folder into the deploy area again. The entire site will update in under 60 seconds.

Method 2 — Netlify CLI (For Developers)

Install the Netlify CLI tool (npm install -g netlify-cli), then from the vaidyagrama-rebuild folder run netlify deploy --prod. Your site updates instantly from the command line.

Method 3 — GitHub Integration (Best for Teams)

Connect Netlify to a GitHub repository containing your website files. Every time you push a file change to GitHub, Netlify automatically detects it and deploys the updated site within 60 seconds. Best for ongoing team collaboration.

Option B — Weebly with HTML Embedding (Limited)

Important Limitation Notice

Weebly is a drag-and-drop website builder and was not designed to host custom multi-page HTML websites. While some content can be embedded, this approach has significant drawbacks: Weebly's own CSS styles will conflict with your custom design, interactive features (Dosha Quiz, Program Finder) may not work, and managing 12 pages this way is time-consuming.

Recommendation: Use Netlify (Option A) and simply update your domain's DNS to point away from Weebly. You can keep your Weebly account dormant or cancel it.

Google Analytics lets you see how many people visit your website, which pages they visit most, where they come from (India, Germany, UK, etc.), how long they stay, and which pages they leave from. This information is essential for improving your website over time.

  1. Go to analytics.google.com and sign in with a Google account. Create a new Property for your website.
  2. Choose "Web" as the platform. Enter your website URL and timezone. You will receive a Measurement ID in the format G-XXXXXXXXXX.
  3. Open each of the 12 HTML files in your text editor. Find the closing </head> tag near the top of each file.
  4. Paste the following code immediately before </head> in all 12 files, replacing G-XXXXXXXXXX with your actual Measurement ID:
<!-- Google Analytics -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>
Time-saving tip: In VS Code, use "Find in Files" (Ctrl+Shift+F) to find </head> across all files, then use "Replace All" to insert the analytics snippet in one step. Be careful to replace the correct closing head tag only.

Key Metrics to Monitor Monthly

Sessions

Total number of visits to your website per month. A healthy baseline for a specialist medical wellness center is 1,000–5,000 sessions/month in year one. Set a target and track growth month-on-month.

Geographic Breakdown

Which countries are your visitors from? Use this to decide which language versions to prioritize (if 20% of visitors are German, the German version becomes urgent).

Top Landing Pages

Which pages do visitors land on first? If the conditions page drives the most organic traffic, it deserves more investment in content and photos.

Backup Instructions

Because your website is a folder of files — not a hosted platform — backing it up is straightforward. There is no database to export, no platform account to worry about. The files on your computer are the source of truth.

Daily Backup (Automatic)

If Netlify is connected to a GitHub repository, every file change creates an automatic version history. You can roll back to any previous version of any file at any time. This is the gold standard.

Monthly Backup (Manual)

Once a month, copy the entire vaidyagrama-rebuild folder to Google Drive or Dropbox. Name the backup folder with the date: vaidyagrama-rebuild-2026-03. Keep at least 3 months of backups.

Before Any Major Edit

Before editing any HTML file, duplicate it first: right-click the file → Copy → Paste → rename the copy to filename_backup_DATE.html. If the edit goes wrong, you can revert instantly.
